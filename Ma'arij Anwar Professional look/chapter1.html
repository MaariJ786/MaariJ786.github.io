    Algorithms

    An algorithm is a sequence of instructions or a set of rules that are followed to complete a task. This task can be anything, so long as you can give clear instructions for it. For example, to make yourself a drink you have to follow a sequence of steps in the right order. If you do something in the wrong order you might end up making a mess. Everyday in life we use an algorithm weather that is to drive or to make breakfast. We do this without even realising. For example we get up have a shower make breakfast then go to work. this is an example of a simple algorithm.

    When you are writing algorithms it is important to remember the 3 key structures in algorithms.They are Iteration , selection and sequence. in order to write a successful algorithm there are two important factors:

    1)They are finite and they are clear instructions

    2)and they are efficient

    Computers won’t understand your algorithm as they use a different language. It will need to be translated into code which the computer will then follow to complete a task. This code is written in a programming language. There are many different types of programming languages. Some that you may come across are Logo, Scratch, Blockly, Python and Kodu. Each of these languages are suited to different things.

    Pseudocode

    Most programs are developed using programming languages. These languages have specific syntax that must be used so that the program will run properly. Pseudocode is not a programming language, it is a simple way of describing a set of instructions that does not have to use specific syntax. Common pseudocode notation

    There is no strict set of standard notations for pseudocode, but some of the most widely recognised are

    INPUT – indicates a user will be inputting something

    OUTPUT – indicates that an output will appear on the screen

    WHILE – a loop (iteration that has a condition at the beginning)

    FOR – a counting loop (iteration)

    REPEAT – UNTIL – a loop (iteration) that has a condition at the end

    IF – THEN – ELSE – a decision (selection) in which a choice is made

    any instructions that occur inside a selection or iteration are usually indented

    Keywords

    An algorithm is a series of step to solve a scenario.

    Iteration: Repetition of steps.

    Selection: A decision step where the computer or the user decides which outcome the algorithm should follow.

    Sub Tasks: A task inside a task.

    Decision: Choice done by the user.

    Process: A series of steps that done an operation.

    Variable: A value that changes throughout the program and is stored in the memory.

    Pseudocode: A sries of steps for the programmer to follow when programing.

    Identifier: A sequence of character used to identify or refer to a program or an element.

    Constant: A value that never changes throughout the program.

    Comment: Written by the programmer to understand the code but ignored by the translator.

    Operator: Characters that represent an action.

    Operand: Values the operation is done on

    Parenthesis: Brackets used separate operations

    Comparison Operator: An operator that compares values to each other. < > <= => ==

    Logical Operator: An operator that checks the state of value.also now as Boolean Operators(AND NOT OR)

    Mathematical Operators: An operator that des mathematical operation on the values.

    Nested IF Statements: IF statements inside IF statements
